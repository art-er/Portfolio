Fo_oD.Style

2020.12-2021.01

This is an individual project created to gain experience in mobile design development. Applications Fo_oD Style aims to combine delivery applications and find recipes applications for the convenience of quarantined users. The goal of the project was to achieve immersion of users by making it familiar and friendly.

Type

Android and iOS app

What makes Food Style unique?

The key aspect for this project was the combination of both regular food delivery and the ability to find many different recipes with step-by-step cooking

Tools

Figma, Adobe XD, Notion, paper, pen and imagination

My role



There was only me on the team

Make the application friendly using all the principles from https://lawsofux.com/

Design Process

1. Storyboard

2. Define

3. Ideate

4. Select

5. Prototype

Storyboard

The storyboard helps us identify key points in the process of ordering / sourcing recipes through a food style app.

  • 1. Enjoy the time

    It was another great day when I just do nothing in quarantine

  • 2. Mom's message

    Damn, mom wrote that the guests are coming soon and I need to make pasta, what should I do?

  • 3. Look for a recipes

    So, I have a Food Style application where I can find a recipe and, if I can't cook, I'll just order ready-made food.

  • 4. Check the fridge

    I found an easy recipe for pasta, it says that you need noodles, minced meat and tomato paste, i need to check the fridge

  • 5. Order ingredients

    It turns out I don't have minced meat, I need to order it and at the same time order pizza and sushi

  • 6. Take order

    All the order came, pretty quickly I didn't even have time to boil the paws

Define

How might i make a more intuitive and simple product for non-tech people?

Problems

  • There are many food delivery and cooking applications on the market
  • Filter ingredients (allergic, vegetarian, halal, ban ingredients)
  • Make the app unique and using features from other similar apps
  • Convenient search for recipes
  • Cluttering Interfaces with too much information on one screen

Ideate

I looked at many different delivery applications and recipes, identifying their shortcomings, created user flows and created lo-fi fidelity wireframes for them with marker and whiteboard, thinking about each step (is it necessary at all, if the user wants to go here, it can be moved and so on). Using lawsofux made the application right

Rethink

The page with the dish and the selection of ingredients has been rethink, it is more convenient to select the number of additional ingredients and it is clear how many ingredients the user has selected or which of them have been removed.
